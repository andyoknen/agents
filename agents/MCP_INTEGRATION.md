# Интеграция через MCP Server

## Обзор

Вместо использования shell команд `cursor-agent`, можно использовать MCP (Model Context Protocol) сервер для более интегрированной работы с Cursor IDE.

## Установка MCP сервера

1. Перейдите в каталог `mcp-server`:
```bash
cd mcp-server
npm install
npm run build
```

2. Настройте Cursor для использования MCP сервера (см. `mcp-server/README.md`)

## Использование в промпте оркестратора

### Старый способ (shell команды):

```markdown
Агентов нужно вызывать shell-командами:
cursor-agent -f --model {модель} -p {промпт}
```

### Новый способ (MCP инструменты):

```markdown
Агентов нужно вызывать через MCP инструменты:
- call_analyst - для аналитика
- call_tz_reviewer - для ревьюера ТЗ
- call_architect - для архитектора
- call_architecture_reviewer - для ревьюера архитектуры
- call_planner - для планировщика
- call_plan_reviewer - для ревьюера плана
- call_developer - для разработчика
- call_code_reviewer - для ревьюера кода

Пример вызова аналитика:
Используй инструмент call_analyst с параметрами:
- task_description: "{постановка задачи}"
- project_description: "{описание проекта (если есть)}"
```

## Преимущества MCP подхода

1. **Интеграция в IDE**: Агенты доступны как нативные инструменты Cursor
2. **Типизация**: Все параметры проверяются через JSON схемы
3. **Удобство**: Не нужно формировать shell команды вручную
4. **Единообразие**: Все агенты вызываются одинаковым способом

## Обратная совместимость

Старый способ с shell командами продолжает работать. MCP сервер - это альтернативный вариант, который можно использовать по желанию.
